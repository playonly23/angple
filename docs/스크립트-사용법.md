# 스크립트 사용법

Angple 프로젝트의 개발/배포를 돕는 스크립트 모음입니다.

## 📋 스크립트 목록

| 스크립트           | 용도                       | 사용 빈도           |
| ------------------ | -------------------------- | ------------------- |
| `server.sh`        | 서버 시작/중지/상태 확인   | 매일                |
| `sync.sh`          | 안전한 코드 동기화         | 매일 (작업 시작 전) |
| `setup-hooks.sh`   | Git hooks 설치             | 최초 1회            |
| `backup-branch.sh` | 작업 내용 백업 브랜치 생성 | 위험한 작업 전      |
| `check-clean.sh`   | Git 상태 검증              | 배포 전             |
| `deploy.sh`        | 프로덕션 배포              | 배포 시             |
| `new-branch.sh`    | 새 브랜치 생성             | 새 작업 시작 시     |
| `finish-branch.sh` | 브랜치 정리                | PR 머지 후          |
| `dev-server.sh`    | dev 서버 관리 (레거시)     | -                   |

## 🚀 server.sh - 서버 통합 관리

dev.damoang.net과 web.damoang.net 서버를 통합 관리합니다.

### 기본 사용법

```bash
./scripts/server.sh {dev|web|all} {start|stop|restart|status}
```

### 예시

```bash
# dev 서버 시작 (dev.damoang.net)
./scripts/server.sh dev start

# web 서버 시작 (web.damoang.net)
./scripts/server.sh web start

# 모든 서버 시작
./scripts/server.sh all start

# dev 서버 재시작 (빌드 + 재시작)
./scripts/server.sh dev restart

# 모든 서버 상태 확인
./scripts/server.sh all status

# web 서버 중지
./scripts/server.sh web stop
```

### 서버 정보

| 서버 | URL             | 포트 | 디렉토리                  | 용도     |
| ---- | --------------- | ---- | ------------------------- | -------- |
| dev  | dev.damoang.net | 3011 | /home/damoang/angple      | 스테이징 |
| web  | web.damoang.net | 3010 | /home/damoang/angple-prod | 프로덕션 |

### 로그 확인

```bash
# dev 서버 로그
tail -50 /tmp/angple-dev.log

# web 서버 로그
tail -50 /tmp/angple-prod.log

# 실시간 로그 모니터링
tail -f /tmp/angple-prod.log
```

## 🔄 sync.sh - 안전한 코드 동기화

로컬 변경사항을 보존하면서 최신 코드를 가져옵니다.

### 기본 사용법

```bash
./scripts/sync.sh [브랜치명]
```

### 예시

```bash
# 현재 브랜치 업데이트 (로컬 변경 보존)
./scripts/sync.sh

# main 브랜치로 전환 후 업데이트
./scripts/sync.sh main
```

⚠️ **--force 옵션은 매우 위험합니다!**

-   로컬의 모든 변경사항이 영구적으로 삭제됩니다
-   복구가 불가능합니다
-   사용하지 마세요! 대신 백업 브랜치를 만드세요:

```bash
# 안전한 방법: 백업 후 초기화
git checkout -b backup-$(date +%Y%m%d-%H%M%S)
git add -A
git commit -m "Backup"
git checkout main
./scripts/sync.sh main
```

### 자동 처리 항목

1. ✅ 로컬 변경사항 임시 저장 (git stash)
2. ✅ 최신 코드 가져오기 (git pull --rebase)
3. ✅ 변경사항 복원 (git stash pop)
4. ✅ 의존성 업데이트 (package.json 변경 감지 시)
5. ✅ 빌드 캐시 정리

## 🪝 setup-hooks.sh - Git Hooks 설치

Git hooks를 설치하여 커밋 전 자동 포맷팅 및 main 브랜치 보호를 활성화합니다.

### 기본 사용법

```bash
./scripts/setup-hooks.sh
```

### 설치되는 Hooks

**pre-commit hook:**

1. **main 브랜치 보호**

    - main/master 브랜치에 직접 커밋 방지
    - 브랜치 생성 안내

2. **자동 코드 포맷팅**
    - 커밋 전 staged 파일만 포맷팅
    - prettier로 자동 정리
    - 포맷팅된 파일 자동 re-stage

### 효과

```bash
# 커밋 시도 시
git commit -m "feat: 새 기능"

# 자동 실행:
# 🔧 코드 포맷팅 중...
# ✅ 포맷팅 완료
# [브랜치명 abc1234] feat: 새 기능
```

### 최초 설정 (권장)

프로젝트를 클론한 후 즉시 실행하세요:

```bash
cd /home/damoang/angple
./scripts/setup-hooks.sh
```

### 이점

-   ✅ CI/CD 포맷 체크 실패 방지 (매번 실패 없음!)
-   ✅ 코드 스타일 자동 통일
-   ✅ main 브랜치 실수 커밋 방지
-   ✅ PR 리뷰 시간 단축

## ✅ check-clean.sh - Git 상태 검증

배포 전 Git 저장소 상태를 검증합니다.

### 기본 사용법

```bash
./scripts/check-clean.sh [--fix]
```

### 예시

```bash
# 상태 확인만
./scripts/check-clean.sh

# 자동 수정 시도 (대화형)
./scripts/check-clean.sh --fix
```

### 검사 항목

1. ✅ 추적되지 않은 파일 (빌드 파일 포함 여부)
2. ✅ 수정된 파일 (커밋 필요 여부)
3. ✅ Staged 파일 (커밋 대기 중)
4. ✅ 원격과 동기화 (push/pull 필요 여부)
5. ✅ data/ 디렉토리 추적 여부

### 배포 전 체크리스트

```bash
# 1. 상태 확인
./scripts/check-clean.sh

# 2. 문제가 있으면 수정
./scripts/check-clean.sh --fix

# 3. 모든 변경사항 푸시
git push origin main

# 4. 배포
./scripts/deploy.sh
```

## 📦 deploy.sh - 프로덕션 배포

angple/에서 빌드 후 angple-prod/에 배포합니다.

### 기본 사용법

```bash
./scripts/deploy.sh
```

### 배포 프로세스

```
[0/4] 배포 전 검증 (check-clean.sh 자동 실행)
  ↓
[1/4] 빌드 (angple/apps/web)
  ↓
[2/4] 파일 복사 (angple/ → angple-prod/)
  ↓
[3/4] 서버 재시작 (web.damoang.net:3010)
  ↓
[4/4] 상태 확인 (HTTP 200 체크)
```

### 안전장치

-   배포 전 Git 상태 자동 검증
-   검증 실패 시 계속 진행 여부 확인
-   기존 서버 graceful shutdown
-   배포 실패 시 로그 경로 안내

### 배포 후 확인

```bash
# 서버 상태 확인
./scripts/server.sh web status

# 로그 확인
tail -50 /tmp/angple-prod.log

# 브라우저 테스트
curl -I https://web.damoang.net
```

## 🌿 new-branch.sh - 새 브랜치 생성

최신 main을 기반으로 새 브랜치를 생성합니다.

### 기본 사용법

```bash
./scripts/new-branch.sh <브랜치명>
```

### 예시

```bash
# 기능 개발 브랜치
./scripts/new-branch.sh feat/사용자인증

# 버그 수정 브랜치
./scripts/new-branch.sh fix/로그인오류

./scripts/new-branch.sh refactor/api클라이언트
```

### 자동 처리

1. main 브랜치로 전환
2. 최신 코드 pull
3. 새 브랜치 생성 및 전환

```bash
./scripts/finish-branch.sh
```

### 자동 처리

## 💾 backup-branch.sh - 작업 백업

위험한 작업(reset, clean 등)을 수행하기 전에 현재 작업 내용을 백업합니다.

### 기본 사용법

```bash
./scripts/backup-branch.sh [백업명]
```

### 예시

```bash
# 자동 이름으로 백업 (backup-YYYYMMDD-HHMMSS)
./scripts/backup-branch.sh

# 커스텀 이름으로 백업
./scripts/backup-branch.sh "실험적기능-백업"
```

### 자동 처리

1. 현재 브랜치에서 새 백업 브랜치 생성
2. 모든 변경사항(tracked + untracked) 커밋
3. 원래 브랜치로 복귀 안내

### 사용 시나리오

#### 시나리오 1: 실험적인 변경 전

```bash
# 백업 생성
./scripts/backup-branch.sh "실험전백업"

# 원래 브랜치로 복귀
git checkout main

# 실험적 작업 수행
# ...

# 실패했으면 백업 복원
git checkout 실험전백업
```

#### 시나리오 2: 복잡한 충돌 발생 시

```bash
# 백업 생성
./scripts/backup-branch.sh

# 원래 브랜치로 복귀 후 초기화
git checkout main
./scripts/sync.sh main

# 나중에 백업에서 필요한 부분만 체리픽
git checkout backup-20260211-143022
git log --oneline  # 필요한 커밋 찾기
git checkout main
git cherry-pick <커밋해시>
```

#### 시나리오 3: 정리가 필요할 때

```bash
# 백업 목록 확인
git branch | grep backup

# 오래된 백업 삭제
git branch -D backup-20260101-120000
```

## 🆘 트러블슈팅

### 서버가 시작되지 않을 때

```bash
# 포트 점유 확인
lsof -i:3010  # web 서버
lsof -i:3011  # dev 서버

# 프로세스 강제 종료
kill -9 $(lsof -t -i:3010)

# 로그 확인
tail -50 /tmp/angple-prod.log

# 재시작
./scripts/server.sh web restart
```

### sync.sh 충돌 발생 시

```bash
# 충돌 파일 확인
git status

# 충돌 해결 후
git add .
git stash drop

# 또는 백업 후 초기화 (안전)
git checkout -b backup-$(date +%Y%m%d-%H%M%S)
git add -A
git commit -m "Backup before reset"
git checkout main
./scripts/sync.sh main
```

### 배포 검증 실패 시

```bash
# 상세 확인
./scripts/check-clean.sh

# 자동 수정 시도
./scripts/check-clean.sh --fix

# 수동 수정 후 재배포
git add -A
git commit -m "chore: 배포 전 정리"
git push origin main
./scripts/deploy.sh
```

### 빌드 파일이 Git에 추적될 때

```bash
# 추적 제거 (파일은 유지)
git rm -r --cached build/ .svelte-kit/
git commit -m "chore: 빌드 파일 추적 제거"

# .gitignore 확인
cat .gitignore | grep -E "build/|\.svelte-kit/"
```
